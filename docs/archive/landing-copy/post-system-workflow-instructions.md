# Client Project Update Post System — Workflow Instructions

> **How to use this file:** This is your working instruction document for mapping out the post system. Fill in the bracketed fields with your actual tools, channels, and specifics. Once complete, this becomes the source of truth for building the flowchart.

---

## Purpose

Define exactly when, what, and how client-facing updates are posted at each stage of a Baltazar Studio project — from Cocoon Consult™ through Winged in a Week™ and into In Full Flight™.

---

## Before You Build the Flowchart

Answer these four questions first. Every node in the flowchart maps back to one of them.

1. **Who sends the update?** (You, an automated system, a tool trigger?)
2. **What triggers it?** (Milestone complete, day of week, client action, internal approval?)
3. **What channel does it go through?** (Email, portal, Notion, Slack, etc.)
4. **What does the update contain?** (Link, summary, next step, approval request?)

## Stage 2 — Winged in a Week™ (Build Phase)

Map updates for each of the seven days:

| Day | Milestone | Update Posted | What Client Sees | Approval Needed? |
|-----|-----------|--------------|-----------------|-----------------|
| Day 1 | Strategy → Page structure | "Build started" update | Sitemap / outline | No |
| Day 2 | Copy locked | Copy draft posted | Copy doc link | Yes — copy approval |
| Day 3 | Design started | Design preview posted | WIP screenshot or link | No |
| Day 4 | Design complete | "Design ready for review" | Figma / staging link | Yes — design approval |
| Day 5 | Development | Build update | Internal only (or optional) | No |
| Day 6 | QA | QA checklist complete | Optional status ping | No |
| Day 7 | Launch-ready | "Your site is ready" | Live preview link | Yes — launch approval |

**Notes / gaps to fill in:**
- [ ] What tool hosts the day-by-day updates? (Notion, client portal, email?)
- [ ] Are updates sent proactively or does the client log in to check?
- [ ] Who handles QA sign-off — you, or does the client review too?

## Flowchart Build Instructions

Once the table above is filled in, use these steps to turn it into a flowchart:

1. **Start node:** Client signs contract
2. **Decision points:** Any row marked "Approval Needed? Yes" becomes a diamond (Yes/No branch)
3. **Action nodes:** Every other row is a rectangle
4. **Channels as swimlanes (optional):** If you use multiple tools (email + Notion + portal), organize the chart by swimlane per channel
5. **End node:** Post-launch system is active / client is self-sufficient

**Suggested tools for the actual flowchart:**
- Figma (FigJam board — recommended for your workflow)
- Whimsical
- Miro
- Notion (inline diagram via Whimsical embed)
